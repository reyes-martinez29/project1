Regla de oro

Haz cambios pequeños y commitea seguido. Un commit debe representar una idea clara: “agregué X” o “corregí Y”.

Flujo típico

  1. 1 Editar archivos
  2. 2 git status
  3. 3 git add .
  4. 4 git commit -m "..."
  5. 5 git push

Tip: si algo sale mal, revisa primero git status.

1) Introducción: Git vs GitHub

Git

Git es un sistema de control de versiones. Te permite guardar “fotografías” (commits) de tu proyecto, comparar cambios, volver atrás y trabajar con ramas sin romper lo que ya funciona.

  • Funciona localmente en tu computadora
  • Te da historial, ramas, merges y herramientas para colaborar

GitHub

GitHub es una plataforma para alojar repositorios Git en la nube y colaborar. Lo importante aquí: repos remotos, Pull Requests, Issues, revisiones y automatización (CI).

  • Tu repo remoto (online)
  • PRs para integrar cambios con revisión
  • Issues para organizar tareas
Idea clave: Git es la herramienta. GitHub es el lugar donde compartes y colaboras.

2) Instalación y configuración inicial

Antes de empezar, asegúrate de tener Git instalado y configurado. Esta configuración aparece en tus commits.

Configurar nombre y correo

Esto firma tus commits. Hazlo una vez.

git config --global user.name "Tu Nombre"
        git config --global user.email "tu_correo@ejemplo.com"

Verificar tu configuración

git config --list

Config útil (opcional)

# Editor para mensajes de commit (VS Code)
        git config --global core.editor "code --wait"

        # Log bonito (atajo)
        git config --global alias.lg "log --oneline --decorate --graph --all"

        # Comprobar tu alias
        git lg

Un alias es solo un atajo para un comando largo. Si algo no te funciona, ejecuta el comando completo.
